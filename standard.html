The current NFy standard

These are what the average NFy Application should have, as more updates come, the spec changes, the Standard will change.

Think of this as a blueprint for the backend of NFy Applications.

Songs - playing

Songs should be allocated one by one, and there is only one song buffer for 1 song.

SongBuffer.play_current()

Songs - Stopping

When stopped (or, while playing) the time will be saved.

When SongBuffer.stopped():
    save_time()

Then, on play if the time isn’t null, load it and play from there.

Songs - Hotkeys

The standard key presses for movement in a song are J K and L.

  • J back 10
  • K Pause
  • L forward 10

Playlists - Shuffle

There’s no current standard for shuffling, it’s not currently required.

Playlists - Iteration

Playlist iteration works like this;

if end of current and in playlist
if next:
    play next

And should be implemented as such. As shown in NFy Mono’s Repository.

Playlists - Format

The standard of playlists has changed since the last version of NFy.

Developers are recommended to switch to playlistAPI v2 instead of v1.

New:

{
    "songs": [

    ]
}

Old:

{
    "playlist_songs": [

    ]
}

simply; for readability and ease of access.

News - Format

Depending on what you’re making, news should ALWAYS be in the format of a date.

When gathering date, you request the date as such; month-day-year.

This saves time, and allows you to download daily news simply. as seen in NFy Mono

News - Website Requirement

News is recommended to be in a news directory. So ease of access is available.

http://mysite.com/news/month-day-year.json

Setup - Directories

Currently, there are two used directories.

  • songs
  • playlists (Rotations in Mono.)
